---
description: Sync task progress to Pull Request comment
argument-hint: <task-id>
---

Sync the progress of task $1 to the corresponding Pull Request comment.

Execute the following steps:

1. Find task file:
   Search in priority order:
   - .ai-workspace/active/$1/task.md (primary)
   - .ai-workspace/blocked/$1/task.md
   - .ai-workspace/completed/$1/task.md
   If none found, prompt that the task does not exist.

2. Read task information:
   Get PR number from task.md (if missing, prompt user to specify manually), title, current_step, status, etc.

3. Read context files (if they exist):
   - analysis.md, plan.md, implementation.md, review.md
   Extract key progress and decisions.

4. Generate progress summary in Markdown:
   ```markdown
   ## 🤖 Development Progress Update

   **Task ID**: $1
   **Updated**: <current time>
   **Current Status**: <status description>

   ### ✅ Completed
   - [x] <completed steps and core points>

   ### 📋 Current Progress
   <current step details>

   ### 🎯 Next Steps
   <next step plan>

   ---
   *Auto-generated by Codex*
   ```

5. Sync to PR:
   ```bash
   gh pr comment <pr-number> --body "<summary content>"
   ```

6. Inform user: show synced PR link and content summary.

**Notes**: Focus on technical decisions and implementation highlights (for reviewers). @mention relevant reviewers.

---
description: Sync task progress to GitHub Issue comment
argument-hint: <task-id>
---

Sync the progress summary of task $1 to the corresponding GitHub Issue comment.

Execute the following steps:

1. Find task file:
   Search in the following priority order:
   - .ai-workspace/active/$1/task.md (priority)
   - .ai-workspace/blocked/$1/task.md
   - .ai-workspace/completed/$1/task.md
   If none found, inform the user the task does not exist.

2. Read task information:
   Get issue_number from task.md (if missing, prompt user to specify manually), title, current_step, status, etc.

3. Read context files (if they exist):
   - analysis.md, plan.md, implementation.md, review.md
   Extract key progress and decisions from them.

4. Generate progress summary:
   Generate a Markdown summary in the following format:
   ```markdown
   ## Task Progress Update

   **Task ID**: $1
   **Updated at**: <current time>
   **Current Status**: <status description>

   ### Completed
   - [x] <completed steps and key points>

   ### Current Progress
   <current step details>

   ### Next Steps
   <next step plan>

   ---
   *Auto-generated by Codex*
   ```

5. Sync to Issue:
   ```bash
   gh issue comment <issue-number> --body "<summary content>"
   ```

6. Inform user:
   - Display the synced Issue link
   - Output summary of synced content

**Notes**:
- Keep summaries concise; extract only key points per phase
- Write for human readers; avoid excessive technical details
- Avoid syncing too frequently; sync after completing a full stage

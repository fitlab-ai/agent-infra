# Sync Issue Command

## Description

Sync a task's progress summary to the corresponding GitHub Issue's comment section, ensuring the Issue contains logical, complete, and trackable information.

## Execution Flow

### 1. Verify Task Exists

Search for the task in the following priority order:
- Check `.ai-workspace/active/{task-id}/task.md` (primary)
- If not found, check `.ai-workspace/blocked/{task-id}/task.md`
- If not found, check `.ai-workspace/completed/{task-id}/task.md`
- If none found, prompt that the task does not exist

Once found, record the task status and task directory path.

Note: `{task-id}` format is `TASK-{yyyyMMdd-HHmmss}`, e.g., `TASK-20260205-202013`

### 2. Read Task Information

Get from the task file:
- Issue number (`issue_number` field)
- Task title and description
- Current step (`current_step`)
- Task status (`status`)
- Created and updated timestamps

### 3. Read Context Files

Check and read the following files (if they exist):
- `.ai-workspace/{status}/{task-id}/analysis.md` - Requirement analysis
- `.ai-workspace/{status}/{task-id}/plan.md` - Technical plan
- `.ai-workspace/{status}/{task-id}/implementation.md` - Implementation report
- `.ai-workspace/{status}/{task-id}/review.md` - Review report

### 4. Generate Progress Summary

Generate a clear progress summary based on the current status:

**Basic format**:
```markdown
## 🤖 Task Progress Update

**Task ID**: {task-id}
**Updated**: {current time}
**Current Status**: {status description}

### ✅ Completed

- [x] Requirement Analysis - {completion time}
  - {Core summary points 1-2 items}
- [x] Technical Design - {completion time}
  - {Solution selection and key decisions 1-2 items}
- [ ] Code Implementation (in progress)
- [ ] Code Review
- [ ] Final Commit

### 📋 Current Progress

{Detailed description of current step}

### 🎯 Next Steps

{Next step plan}

### 📂 Related Files

- Task file: `.ai-workspace/{status}/{task-id}/task.md`
- Requirement analysis: `.ai-workspace/{status}/{task-id}/analysis.md`
- Technical plan: `.ai-workspace/{status}/{task-id}/plan.md`

---
*Auto-generated by Claude Code - [Task Management System](../.agents/README.md)*
```

**Summary principles**:
- **Concise**: Extract only core points for each phase, avoid being verbose
- **Logical flow**: Show progress in chronological order
- **Highlight key decisions**: Solution selections, important findings, etc.
- **Human-readable**: Avoid technical details, use easy-to-understand language

### 5. Sync to Issue

Use `gh` command to post the summary to the Issue:

```bash
gh issue comment {issue-number} --body "$(cat <<'EOF'
{generated progress summary}
EOF
)"
```

### 6. Update Task Status

Record the sync time in the task file:
- Add or update `last_synced_at` field
- Record the Issue comment link (if returned by gh)

### 7. Inform User

Output format:
```
Task progress synced to Issue #{issue-number}

**Synced Content**:
- Completed steps: {count}
- Current status: {status}
- Next step: {next step description}

**View Link**:
https://github.com/{owner}/{repo}/issues/{issue-number}
```

## Parameters

- `<task-id>`: Task ID, format TASK-{yyyyMMdd-HHmmss} (required)

## Usage Example

```bash
# Sync task progress to corresponding Issue
/sync-issue TASK-20251227-104654
```

## Notes

1. **Issue Number Must Exist**:
   - Task file must have an `issue_number` field
   - If not, prompt user to manually specify or update the task file

2. **Summary Generation Principles**:
   - Written for project managers and other developers
   - Highlight key decisions and progress
   - Avoid excessive technical details
   - Maintain logical clarity

3. **Sync Timing**:
   - After completing important phases (analysis, design, implementation, review)
   - When encountering blocking issues
   - Periodic updates for long-running tasks

4. **Comment Format**:
   - Use Markdown formatting
   - Use emoji for readability
   - Include timestamps
   - Add Claude Code signature

5. **Avoid Frequent Syncing**:
   - Don't sync at every small step
   - Recommend syncing after completing a full phase
   - Avoid generating too many comments

## Error Handling

- Task not found: Prompt "Task {task-id} not found, please check the task ID"
- Missing Issue number: Prompt "Task file is missing the issue_number field"
- gh command failed: Prompt "Sync failed, please check if GitHub CLI is authenticated"
- Network error: Prompt "Network connection failed, please try again later"
